## 一、C++对C的核心扩展（基础必考：选择/填空/简答）
### 核心知识点
1. 输入输出：`cin/cout` 替代`scanf/printf`，无需格式符，头文件`<iostream>`，`using namespace std;`
2. 引用：`&`定义，变量别名，**必须初始化**，本质是安全指针
3. const常量：替代`#define`，必须初始化，类型安全
4. new/delete：替代`malloc/free`，自动计算大小、初始化、类型安全
5. 函数重载：同一作用域，同名函数，**参数不同**（个数/类型/顺序）
6. 默认参数：从右往左赋值，声明/定义只能一处写默认值
7. 内联函数：`inline`修饰，减少调用开销，不能有循环/递归/复杂语句
8. bool类型：`true(非0)/false(0)`，C++独有

### 重点
- 引用 vs 指针：引用是别名，无空引用，自动解引用；指针是地址，可空、可修改
- **new/delete 与 malloc/free 区别**（笔试必考简答）
- 函数重载的判断规则

### 🔴 难点
- 引用作为函数返回值（不能返回局部变量引用）
- 默认参数与函数重载的二义性

### ⚠️ 易错点
1. 引用**必须初始化**，不能单独定义
2. `new/delete` 严格配对：`new`单个→`delete`；`new[]`数组→`delete[]`
3. 函数重载**仅返回值不同不算重载**
4. 默认参数必须**从右往左**设置，不能中间跳跃
5. `cout`打印指针：`char*`打印字符串，其他指针需强转`(void*)`

## 二、类与对象（分值最高：选择/填空/代码/改错）
### 核心知识点
1. 封装：私有`private`（默认）、保护`protected`、公有`public`
2. 构造函数：与类同名，无返回值，自动调用，用于初始化
3. 析构函数：`~类名()`，无参无返回值，自动调用，释放资源
4. this指针：隐含指针，指向当前对象，解决成员与参数同名问题
5. 初始化列表：`类名(参数):成员1(值),成员2(值){}`
6. 对象创建：栈对象（自动析构）、堆对象（`new`创建/`delete`释放）
7. 对象数组/指针：数组初始化、指针访问成员`->`

### 重点（笔试100%考）
1. 构造/析构调用顺序：构造**先父后子**，析构**先子后父**；栈对象**后进先出**
2. this指针使用：成员与参数同名、返回对象本身`return *this`
3. 初始化列表**必须用**场景：const成员、引用成员、无默认构造的对象成员
4. 拷贝构造函数：`类名(const 类名& obj)`，浅拷贝/深拷贝

### 🔴 难点
**深拷贝与浅拷贝**（MyString作业核心，改错/代码必考）
- 浅拷贝：仅拷贝指针地址，多对象共享内存，析构重复释放→崩溃
- 深拷贝：重新分配内存，拷贝内容，解决浅拷贝问题

### ⚠️ 易错点
1. 构造函数**无返回值**，不能写`void`
2. 析构函数**无参、不能重载**
3. 同名成员赋值：`this->成员 = 参数`
4. 堆对象`new`后必须`delete`，否则内存泄漏
5. 初始化列表**先于构造函数体执行**，const成员只能用列表初始化
6. 默认对象赋值是浅拷贝，动态内存类必须重写拷贝构造

## 三、继承与派生（高频考点：选择/简答/改错）
### 核心知识点
1. 继承语法：`class 派生类:继承方式 基类{}`
2. 三种继承方式（必考表格）

| 继承方式 | 基类public | 基类protected | 基类private |
|----------|------------|--------------|-------------|
| public（常用） | 仍public | 仍protected | 不可访问 |
| protected | 变protected | 仍protected | 不可访问 |
| private | 变private | 变private | 不可访问 |

3. 派生类构造函数：必须调用基类构造函数，`派生类(参数):基类(参数){}`
4. 菱形继承：二义性问题，**虚基类**解决：`virtual public 基类`
5. 向上转型：基类指针/引用指向派生类对象，安全

### 重点
1. 三种继承方式的权限变化
2. 派生类构造函数写法（必须初始化基类）
3. 虚基类解决菱形继承二义性

### 🔴 难点
菱形继承原理与虚基类实现机制

### ⚠️ 易错点
1. 基类**private成员**，任何继承都**不能直接访问**
2. 基类无默认构造时，派生类必须**显式调用基类构造**
3. 向上转型后，非虚函数只能调用基类版本
4. 虚基类`virtual`位置：`class 派生类: virtual public 基类`

## 四、动态多态性（最难考点：选择/简答/代码/改错）
### 核心知识点
1. 动态多态：运行时确定调用函数，**基类指针/引用指向派生类**
2. 实现三要素（必考简答）
   - 公有继承
   - 基类定义**虚函数**（`virtual`）
   - 派生类**重写**虚函数
3. 虚函数：`virtual 返回值 函数名(参数){}`
4. 纯虚函数：`virtual 返回值 函数名(参数)=0`
5. 抽象类：含纯虚函数，**不能创建对象**，派生类必须重写
6. override/final：`override`检查重写，`final`禁止重写/继承
7. 虚析构函数：基类析构加`virtual`，避免内存泄漏

### 重点（压轴考点）
1. 动态多态实现条件
2. 虚函数作用
3. 纯虚函数与抽象类
4. 虚析构函数必要性

### 🔴 难点
虚函数表（vtable）与虚指针（vptr）底层原理

### ⚠️ 易错点
1. **构造函数不能是虚函数**，析构函数最好为虚函数
2. 虚函数重写：**函数名、参数、返回值必须完全一致**
3. 抽象类**不能实例化**
4. 基类指针delete派生类，析构非虚→只调基类析构→泄漏
5. `override`必须加在派生类重写函数后

## 五、运算符重载（必考：代码/填空/改错）
### 核心知识点
1. 本质：函数重载，`返回值 operator运算符(参数){}`
2. 可重载：`+ - * / == << >> [] =`等
3. 不可重载：`:: .* sizeof ?: `等
4. 友元重载：`<< / >>`必须用友元（左操作数是流）
5. 成员重载：双目运算符**1个参数**，单目**0个参数**
6. 赋值运算符`=`：默认浅拷贝，动态内存类必须深拷贝重载

### 重点（作业核心）
1. 输入输出`<< / >>`友元重载
2. 算术`+ / +=`重载
3. 关系`==`重载
4. 赋值`=`深拷贝重载

### 🔴 难点
运算符重载返回值设计：`+=`返回引用，`+`返回对象

### ⚠️ 易错点
1. `<< / >>`必须用友元，不能用成员
2. 双目成员重载**仅1个参数**（左操作数是this）
3. 赋值重载需判断**自赋值**：`if(this==&obj) return *this`
4. 重载不改变优先级、结合性、操作数个数

## 六、模板（填空/简答）
### 核心知识点
1. 函数模板：`template<typename T> 返回值 函数名(T 参数){}`
2. 类模板：`template<typename T1,typename T2> class 类名{}`
3. 作用：泛型编程，代码复用，支持多类型

### 重点
类模板定义与使用、模板实例化`类名<类型> 对象`

### ⚠️ 易错点
1. 模板声明+实现**必须同文件**，不能分.h/.cpp
2. 类模板成员类外实现：`template<typename T> 返回值 类名<T>::函数名(){}`

## 七、STL基础：vector（填空/代码）
### 核心知识点
1. 动态数组，头文件`<vector>`
2. 常用操作
   - 创建：`vector<int> v;` `vector<int> v(5);` `vector<int> v={1,2,3};`
   - 添加：`push_back()` `emplace_back()`
   - 访问：`v[i]` `v.at(i)`（越界抛异常）
   - 大小：`size()` `resize()` `capacity()`
   - 删除：`pop_back()` `erase(迭代器)`
3. 遍历：普通for、范围for`for(auto& x:v)`

### 重点
vector动态扩容、常用成员函数、迭代器使用

### ⚠️ 易错点
1. `v[i]`越界**不报错**，`v.at(i)`越界抛异常
2. `erase`后迭代器失效
3. 存指针需手动`delete`释放

## 八、静态成员与友元（选择/填空）
### 核心知识点
1. 静态成员
   - 静态数据成员：`static 类型 成员`，**类内声明，类外初始化**
   - 静态成员函数：`static 返回值 函数名(){}`，无this，仅访问静态成员
2. 友元
   - 友元函数：`friend 函数声明`
   - 友元类：`friend class 类名`

### 重点
静态成员访问`类名::成员`、友元访问私有成员

### ⚠️ 易错点
1. 静态数据成员**必须类外初始化**
2. 静态函数**不能访问非静态成员**
3. 友元**无传递性、无继承性**

## 九、内存管理（简答/改错）
### 核心知识点
1. RAII：构造分配，析构释放，管理动态内存
2. new/delete：分配+构造 / 析构+释放
3. 内存泄漏：动态内存未释放

### 重点
MyString类动态内存管理（深拷贝、析构、set重分配）

### ⚠️ 易错点
1. `new[]`必须`delete[]`，严格配对
2. 浅拷贝→重复释放→崩溃
3. 释放后置`nullptr`，避免野指针
